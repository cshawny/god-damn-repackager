# mcmod 模组介绍（直接复制到 mcmod.cn 的模组词条）

## 名称
并行理包

## 英文名
God Damn Repackager

## 简介（一句话，用于词条摘要）
让同一保险库上的多个理包机并行处理一张大订单，N 个理包机 ≈ N 倍合成速度。

## 详细介绍

### 这个模组解决什么问题？

在原版机械动力 6.0+ 中，当你通过仓库管理员（Stockkeeper）对一个合成器阵列下达大批量合成订单时（比如"合成 1000 个铁块"），**整张订单只会被一个理包机处理**。即使你的输入保险库四周贴满了理包机，也只有其中一个在忙活，其余全部闲置。

理包机每秒只能输出一个包裹（动画周期 20 ticks），所以大订单要等很久。

**并行理包**让同一保险库上的所有理包机并行分担这张订单——队列更短的理包机分到更多活，所有机器的产能都被利用。

### 效果对比

| 场景 | 原版 | 加上本模组 |
|---|---|---|
| 1000 个合成，1 个理包机 | 约 1000 秒 | 约 1000 秒（无变化） |
| 1000 个合成，3 个理包机 | 约 1000 秒（其余 2 个闲置） | 约 333 秒 |
| 1000 个合成，9 个理包机 | 约 1000 秒（其余 8 个闲置） | 约 111 秒 |

### 使用方法

**不需要任何额外操作，开箱即用。** 像平时一样搭合成器阵列即可：

```
总仓库 ──订单──> 蛙港发货 ──> 输入保险库(装着原料包裹)
                                   ↓
                        多个理包机(贴红石块常开)
                                   ↓
                        打包机(拆包) → 动力合成器 → 产物
```

只要输入保险库周围有多个理包机，本模组就会自动让它们并行工作。理包机必须贴在机械动力的保险库（Vault）上。

### 兼容性

- 已测试：Minecraft 1.20.1 + Forge 47.x + Create 6.0.8
- 已在含其它模组的整合包和多人服务器中测试，无冲突
- 仅针对 Create 6.0.x 物流系统，不兼容 0.5.1 及更早版本
- 仅 Forge 版本

### ⚠️ 版本与风险说明

当前版本 **0.2.0 为 alpha 测试版**。本模组通过 Mixin 修改机械动力的理包机核心逻辑，尚未经过大规模长期稳定性测试。**使用前请务必备份存档。** 如遇 bug 欢迎反馈。
