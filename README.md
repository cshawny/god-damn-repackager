A Forge mod for Minecraft 1.20.1 that makes Create 6.0+'s Repackagers parallelize a single large
crafting order across all repackagers sharing the same input vault.

---

## 这个模组解决什么问题

在原版机械动力 6.0+ 中，当你通过仓库管理员（Stockkeeper）对一个合成器阵列下达一个大批量合成订单时
（例如"合成 1000 个铁块"），**整张订单只会被一个理包机（Repackager）处理**。即使你的输入保险库
周围贴满了理包机，也只有其中一个在忙活，其余的全部闲置。一个理包机每秒只能输出一个包裹（动画周期
20 ticks），所以大订单要等很久。

本模组让**同一保险库上的所有理包机并行分担**这张订单。N 个理包机 ≈ N 倍的处理速度。

### 效果对比

| 场景 | 原版 | 加上本模组 |
|---|---|---|
| 1000 个合成，1 个理包机 | ~1000 秒 | ~1000 秒（无变化） |
| 1000 个合成，3 个理包机 | ~1000 秒（其余 2 个闲置） | ~333 秒 |
| 1000 个合成，9 个理包机 | ~1000 秒（其余 8 个闲置） | ~111 秒 |

## 工作原理（简述）

理包机在整理订单时会一次性把整张订单的合成包裹塞进**自己的**发送队列，然后每秒发出一个。
本模组用两层机制让所有理包机并行分担：

1. **快照分配**：拦截"塞进队列"的动作，把包裹按当前队列深度负载均衡地分给同一保险库上的
   所有理包机——队列更短的分到更多活。
2. **动态再平衡**（0.3.0 新增）：每隔约 0.5 秒，如果某台理包机积压严重而另一台空闲，会把积压
   方队尾的一部分包裹匀给空闲方。这样即使某台理包机的下游通道堵塞，它的活也会被空闲兄弟动态
   分担，总体吞吐不受单台拖累。

详细的技术实现见 [TECHNICAL.md](TECHNICAL.md)。

## 安装

1. 安装 **Minecraft 1.20.1 + Forge 47.x**
2. 安装 **Create 6.0.x**（已测试 6.0.8）
3. 把 `goddamnrepackager-0.3.0-alpha.jar` 放进 `.minecraft/mods/` 文件夹
4. 启动游戏

> **从 0.2.0 升级？** 直接替换 jar 即可。0.2.0 曾要求在老存档里重新放置理包机，
> 0.2.1 已修复此问题，升级后无需任何额外操作。详见下方[已知限制](#已知限制)。

不需要任何配置，开箱即用。

## 使用方法

**不需要任何额外操作。** 像平时一样搭你的合成器阵列：

```
[总仓库] ──订单──> 蛙港发货 ──> 输入保险库(装着原料包裹)
                                       ↓
                            多个理包机(贴红石块常开)
                                       ↓
                            打包机(拆包) ──> 动力合成器 ──> 产物
```

只要输入保险库周围有多个理包机，本模组就会自动让它们并行工作。

## 已知限制

- ~~**安装到已存在的存档后，需要重新放置理包机。**~~ **（0.2.1 已修复）** 0.2.0 在老存档
  （尤其多人服务器）里可能出现"下单后只有部分理包机工作"（例如 9 个只有 6 个工作），当时需要
  手动重新放置理包机。根因是 sibling 判定用了脆弱的 `IItemHandler ==`（能力实例代次敏感）。
  **0.2.1 改用 Create 的 `InventoryIdentifier` 值相等判定同库，此问题不再出现，升级即可，无需
  重新放置。** 技术细节见 [TECHNICAL.md §3.7](TECHNICAL.md)。
- ~~当前实现是"一次性快照分配"，分配完就静态，空闲理包机不会去抢别人的活。~~
  **（0.3.0 已改善）** 0.3.0 在快照分配之上叠加了**动态再平衡**：每隔约 0.5 秒检查各理包机队列，
  把积压方的部分包裹匀给空闲方。这样即使某台理包机下游堵塞，它的活也会被空闲兄弟分担。
  再平衡严格保护订单顺序（只动队尾、保守阈值），不会打乱合成。技术细节见
  [TECHNICAL.md §2.2④](TECHNICAL.md)。
- 理包机必须贴在**机械动力的保险库（Vault）**上。其它容器（板条箱、原版箱子）理论上也支持，
  但尚未充分测试。

## 调试

如果需要排查问题，可以在 `GodDamnRepackager.java` 里把 `DEBUG_LOGGING = true` 重新编译。
开启后每次订单分配都会在日志里打印 `[GDR-DIST]` 行，显示每个理包机分到了多少包裹。
